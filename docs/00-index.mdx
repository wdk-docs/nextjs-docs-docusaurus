欢迎来到Next.js文档。

## Next.js是什么?

Next.js 是一个构建web应用程序的框架。

使用Next.js，你可以使用React组件构建用户界面。然后，Next.js为您的应用程序提供了额外的结构、特性和优化。

在底层，Next.js还为您抽象和自动配置工具，比如捆绑、编译等等。这使您可以专注于构建应用程序，而不是花时间设置工具。

无论你是个人开发人员还是大型团队的一员，Next.js都可以帮助你构建交互式、动态和快速的web应用程序。

## 主要特点

Next.js的一些主要特性包括:

| 功能                                                                     | 描述                                                                                                                             |
| ------------------------------------------------------------------------ | -------------------------------------------------------------------------------------------------------------------------------- |
| [Routing](/docs/app/building-your-application/routing)                   | 一个基于文件系统的路由器，建立在服务器组件之上，支持布局、嵌套路由、加载状态、错误处理等。                                       |
| [Rendering](/docs/app/building-your-application/rendering)               | 使用客户端和服务器组件的客户端和服务器端渲染。使用Next.js在服务器上进一步优化了静态和动态渲染。在Edge和Node.js运行时上流式传输。 |
| [Data Fetching](/docs/app/building-your-application/data-fetching)       | 在React组件中使用async/await支持简化了数据获取，并且`fetch()` API与React和Web平台保持一致。                                      |
| [Styling](/docs/app/building-your-application/styling)                   | 支持您喜欢的样式方法, including CSS Modules, Tailwind CSS, and CSS-in-JS                                                         |
| [Optimizations](/docs/app/building-your-application/optimizing)          | 图像，字体和脚本优化，以提高您的应用程序的核心网络生命和用户体验。                                                               |
| [TypeScript](/docs/app/building-your-application/configuring/typescript) | 改进了对类型脚本的支持，具有更好的类型检查和更有效的编译，以及自定义类型脚本插件和类型检查器。                                   |
| [API Reference](/docs/app/api-reference)                                 | 更新了整个Next.js的API设计。有关新APIs，请参阅API参考部分。                                                                      |

## 如何使用这些文档

文档的部分和页面是按顺序组织的，从基本到高级，因此在构建Next.js应用程序时可以一步一步地遵循它们。但是，您可以按任何顺序阅读它们，或者跳到适用于您的用例的页面。

在侧边栏的顶部，你会注意到一个下拉菜单，允许你在 **App Router** 和 **Pages Router** 功能之间切换。由于每个目录都有独特的特性，因此跟踪选择了哪个选项卡非常重要。

在页面的右侧，您将看到一个目录，可以更容易地在页面的各个部分之间导航。页面顶部的面包屑还会指示你是在查看 **App Router** 文档还是 **Pages Router** 文档。

要开始，签出[安装](/docs/getting-started/installation)。
如果你是React或服务器组件的新手，我们建议你阅读[React 概要](/docs/getting-started/react-essentials)页面。

为了充分利用我们的文档，建议您对HTML、CSS和React有基本的了解。如果你需要提高你的React技能，看看这些资源:

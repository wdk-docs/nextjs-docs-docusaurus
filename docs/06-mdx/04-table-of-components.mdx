# 组件

MDX的一大优点是，您可以编写标记并指定要使用的组件，而不是HTML元素。
下表列出了那些可以覆盖的HTML元素、所需的标记以及正常的JSX等效元素。
但是，以`a`的第一行为例，您可以通过以下操作将该超链接替换为`<FancyLink>`:

```jsx
import Readme from './readme.md' // Assumes an integration is used to compile MDX -> JS.
import { FancyLink } from './components/fancy-link.js'
;<Readme components={{ a: FancyLink }} />
```

更多关于如何使用markdown的信息可以在[CommonMark][]中找到。

## 总结

可以覆盖的组件使用它们的标准HTML名称。
通常，在markdown里，这些名字是: `a`, `blockquote`, `br`, `code`, `em`,
`h1`, `h2`, `h3`, `h4`, `h5`, `h6`, `hr`, `img`, `li`, `ol`, `p`, `pre`,
`strong`, 和 `ul`.

如果使用 [`remark-gfm`][gfm] ([查看指南][guide-gfm]), 你也可以用: `del`,`input`, `section`, `sup`, `table`, `tbody`, `td`, `th`, `thead`, 和 `tr`。

其他添加了对新结构的支持并宣称它们可以与rehype一起工作的remark插件也可以与MDX一起工作。

更多关于组件的信息，请参见[在§使用MDX里的¶组件][components]。

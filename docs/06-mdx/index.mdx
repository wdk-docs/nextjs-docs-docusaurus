# MDX

**组件 era** 的 Markdown

MDX允许在您的Markdown内容中使用JSX。
您可以导入组件，如交互式图表或警报，并将它们嵌入到内容中。
这使得编写带有组件的长篇内容成为一种冲击。

## MDX是做什么的?

你可以通过JSX用嵌入的组件来写Markdown:

```mdx title="example.mdx"
import { Chart } from './snowfall.js'
export const year = 2018

# Last year’s snowfall

In {year}, the snowfall was above average.
It was followed by a warm spring which caused
flood conditions in many of the nearby rivers.

<Chart year={year} color="#fcb32c" />
```

它被编译成JavaScript，你可以在任何支持JSX的框架中使用:

```md
{/_ lint disable _/}

<div className="home-preview">
  # Last year’s snowfall

In {year}, the snowfall was above average.
It was followed by a warm spring which caused flood conditions in many of
the nearby rivers.

  <Chart year={year} color="#fcb32c" />
</div>

```

## 简称MDX

- ❤️ **强大的**: MDX混合了Markdown和JSX语法，非常适合基于JSX的项目
- 💻 **一切都是组成部分**: 使用MDX中的现有组件，并导入其他MDX文件作为组件
- 🔧 **可定制的**: 决定为每个标记减少构造呈现哪个组件 (`{h1: MyHeading}`)
- 📚 **Markdown基础**: Markdown的简单性和优雅性仍然存在，您只在需要时才使用JSX
- 🔥 **飞快地**: MDX没有运行时，所有编译都在构建阶段进行

> 它对于使用设计系统组件来呈现Markdown和将交互组件与现有Markdown编织在一起非常有用。

Next.js可以提供静态文件，比如图片，放在根目录下一个名为`public`的文件夹下。`public`中的文件可以由你的代码从基础URL`/`)开始引用。

例如，如果你在`public`中添加`me.png`，下面的代码将访问图像:

```jsx title="Avatar.js"
import Image from 'next/image'

export function Avatar() {
  return <Image src="/me.png" alt="me" width="64" height="64" />
}
```

### PagesOnly

此文件夹也适用于`robots.txt`, `favicon.ico`， Google Site Verification和任何其他静态文件(包括`.html`)。
但是请确保没有一个静态文件与`pages/`目录中的文件同名，因为这会导致错误。
[阅读更多](/docs/messages/conflicting-public-file-page).

### AppOnly

对于静态元数据文件，如`robots.txt`, `favicon.ico`等，你应该使用`app`文件夹中的[特殊元数据文件](/docs/app/api-reference/file-conventions/metadata)。

:::tip

- 目录必须命名为`public`。名称不能更改，它是用于提供静态资产的唯一目录。
- 只有在[构建时](/docs/app/api-reference/next-cli#build)的`public`目录下的资产才会被Next.js提供服务。
  在运行时添加的文件将不可用。
  我们建议使用第三方服务，如[AWS S3](https://aws.amazon.com/s3/)进行持久文件存储。

:::

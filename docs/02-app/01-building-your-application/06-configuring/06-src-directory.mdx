除了在项目的根目录中放置特殊的Next.js`app` 或 `pages` 目录外，Next.js还支持将应用程序代码放在`src`目录下的常见模式。

这将应用程序代码从项目配置文件中分离出来，这些文件大多位于项目的根目录中。这是一些个人和团队的首选。

要使用`src`目录，请将`app` Router文件夹或`pages` Router文件夹分别移动到`src/app` 或 `src/pages` 。

> **注意事项**
>
> - `/public`目录应该保留在项目的根目录中。
> - 像`package.json`， `next.config.js`和`tsconfig.json`这样的配置文件应该保留在项目的根目录中。
> - `.env.*` 文件应保留在项目的根目录中。
> - `src/app` 或 `src/pages`如果`app` 或 `pages`存在于根目录中，将被忽略。
> - 如果你使用`src`，你可能还会移动其他应用程序文件夹，如`/components`或`/lib`。
> - 如果你使用顺风CSS，你需要在[content部分](https://tailwindcss.com/docs/content-configuration)的`tailwind.config.js`文件中添加`/src`前缀。

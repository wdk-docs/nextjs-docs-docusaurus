开始使用Next.js最简单的方法是使用`create-next-app`。
这个CLI工具使您能够快速开始构建新的Next.js应用程序，并为您设置好一切。

你可以使用默认的Next.js模板创建一个新的应用程序，或者使用一个[官方Next.js示例](https://github.com/vercel/next.js/tree/canary/examples).

### 互动

您可以通过运行以下命令交互式地创建一个新项目:

```bash title="Terminal"
npx create-next-app@latest
```

```bash title="Terminal"
yarn create next-app
```

```bash title="Terminal"
pnpm create next-app
```

然后，系统会向您询问以下提示:

```bash title="Terminal"
What is your project named?  my-app
Would you like to use TypeScript?  No / Yes
Would you like to use ESLint?  No / Yes
Would you like to use Tailwind CSS?  No / Yes
Would you like to use `src/` directory?  No / Yes
Would you like to use App Router? (recommended)  No / Yes
Would you like to customize the default import alias?  No / Yes
```

一旦您回答了提示，将根据您的回答创建一个具有正确配置的新项目。

### 非交互式

还可以通过命令行参数以非交互方式设置新项目。

此外，你可以通过在默认选项前加上`--no-`来否定它们。 (如 `--no-eslint`).

查看 `create-next-app --help`:


### 为什么使用Create Next App?

`create-next-app` 允许你在几秒钟内创建一个新的Next.js应用程序。它由Next.js的创建者正式维护，并包括许多好处:

- **互动体验**: 运行`npx create-next-app@latest`(不带参数)会启动一个交互式体验，指导您设置项目。
- **零依赖关系**: 初始化项目只需一秒钟。Create Next App没有依赖项。
- **离线支持**: Create Next App会自动检测你是否离线，并使用本地包缓存引导你的项目。
- **示例支持**: Create Next App可以使用Next.js示例集合中的示例来引导应用程序(例如:`npx create-next-app --example api-routes`)或任何公共GitHub存储库。
- **测试**: 该包是Next.js单包的一部分，并使用与Next.js本身相同的集成测试套件进行测试，以确保它在每个版本中都能按预期工作。

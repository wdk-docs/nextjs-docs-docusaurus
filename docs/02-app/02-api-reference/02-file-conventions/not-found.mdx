**not-found**文件用于在路由段中抛出[`notFound`](/docs/app/api-reference/functions/not-found)函数时呈现UI。除了提供自定义UI外，Next.js还将返回一个`404`HTTP状态码。

```tsx title="app/blog/not-found.tsx" switcher
import Link from 'next/link'

export default function NotFound() {
  return (
    <div>
      <h2>Not Found</h2>
      <p>Could not find requested resource</p>
      <p>
        View <Link href="/blog">all posts</Link>
      </p>
    </div>
  )
}
```

```jsx title="app/blog/not-found.js" switcher
import Link from 'next/link'

export default function NotFound() {
  return (
    <div>
      <h2>Not Found</h2>
      <p>Could not find requested resource</p>
      <p>
        View <Link href="/blog">all posts</Link>
      </p>
    </div>
  )
}
```

> **注意事项**: 除了捕获预期的`notFound()`错误外，根`app/not-found.js`文件还处理整个应用程序的任何不匹配的url。这意味着用户访问的URL不是你的应用程序处理的，将显示由`app/not-found.js`文件导出的UI。

## 属性

`not-found.js` 组件不接受任何属性。
